# 模型方法运行详解

## 接口地址

```
POST {host}/xcodegw/app/reg4bc6558503724/tag/{env}/api/run/odexftopenapiv2appmodelmethodrun?appTag={env}&modelKey={key}&methodKey={key}
```

| 环境 | 地址 |
|------|------|
| UAT | `https://xft-demo.cmburl.cn/.../tag/uat/api/run/...` |
| PRD | `https://xft.cmbchina.com/.../tag/prd/api/run/...` |

## 请求方式

- Method: POST
- Content-Type: application/json
- credentials: include

## 入参规则

> [!important] 核心规则
> 每个模型方法的入参 JSON key 必须与该方法的 SQL 定义中 `#{xxx}` 占位符完全匹配。
> 例如 SQL 为 `WHERE sn_code = #{sn_code}`，则入参必须包含 `"sn_code"` 字段。

> [!warning] 前端参数转换
> - 前端传 camelCase（如 `orderNo`），通过 `convertParamsToSnakeCase` 自动转 snake_case（`order_no`）
> - `pageSize` 和 `current` 保留不转
> - `returnCode` 在 `src/api/request.js` 中归一化为 `code`

## 通用分页入参

```json
{
  "current": 1,
  "pageSize": 20
}
```

## 通用查询排序

```json
{
  "orders": [
    { "key": "id", "order": "DESC" }
  ]
}
```

## 响应格式

```json
{
  "returnCode": "SUC0000",
  "errorMsg": null,
  "body": {
    "list": [...],
    "total": 100,
    "current": 1,
    "pageSize": 20
  }
}
```

## 常见错误

| 现象 | 原因 | 解决 |
|------|------|------|
| 接口返回 200 但 data 为空 | 入参字段名不匹配 SQL | 核对 MODEL_REFERENCE.md 中 SQL 字段 |
| 分页无效 | current/pageSize 未传递 | 确保 pageSize 有默认值 |
| 中文乱码 | 文件非 UTF-8 | 转为 UTF-8 无 BOM |
| SN 查询不到 | status 字段名写成了 snStatus | 字段名统一为 status |
