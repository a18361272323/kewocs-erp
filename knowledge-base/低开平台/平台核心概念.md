# 薪福通低代码平台核心概念

## 平台架构

> [!info] 运行模式
> 我们的 Vue 项目通过 iframe srcdoc 嵌入薪福通页面，静态资源从 Cloudflare CDN 加载。
> API 请求发往薪福通同域，Cookie 自动携带（credentials: include）。

## 六大核心概念

### 1. 应用（App）

- 一个独立的业务系统，对应 
eg4bc6558503724
- 有独立的模型、接口、页面、权限配置
- 环境隔离：UAT（xft-demo.cmburl.cn）vs PRD（xft.cmbchina.com）

### 2. 模型（Model）

- 数据表定义，托管在平台后端数据库（MySQL）
- 每个模型有唯一 modelKey，如 MOk2ZJ4aga（SN码表）
- 支持短文本、长文本、数字、日期、下拉等字段类型
- 短文本字段支持正则校验、唯一性校验
- 共 21 个业务模型，详见 [[../项目架构/模型速查]]

### 3. 模型方法（Model Method）

- 对模型的 CRUD 操作，通过 methodKey 调用
- 如 FUG5LjJIRx = SN码列表查询
- 每个方法的 SQL 定义决定了入参和出参结构
- 详见 [[模型方法运行]]

### 4. 接口（API）

- HTTP 接口：对接外部系统
- 业务编排：可视化后端逻辑编排
- SQL 脚本节点：直接执行 SQL
- 详见 [[HTTP接口与业务编排]]

### 5. 组件（Component）

- 页面 UI 构建块：Table、Form、Select、CardList 等
- 我们项目使用 Element Plus，不依赖低开组件系统
- 低开组件的文档作为参考

### 6. 变量系统

- xcInput / xcOutput：接口适配传参
- xcHeader / xcQuery：读写请求头/查询参数
- ctxData：组件间数据联动
- xcUser / xcAuth：用户信息和权限
- 详见 [[变量系统详解]]
