# 开发规范

## 文件编码

> [!important] 必须 UTF-8 无 BOM
> 所有 .vue/.js/.html 文件统一 UTF-8 无 BOM 编码。
> GBK 或 UTF-16 LE 在 Cloudflare Linux 构建环境会解析失败。

## API 调用模式

```javascript
// 统一通过 src/api/index.js 导出
import { snApi, stockInApi, stockOutApi } from '@/api'

// 始终检查 code
if (res.code === 'SUC0000') {
  // 成功处理
}

// 错误处理
catch (error) {
  console.error('...', error)
  ElMessage.error('...')
}
```

## 字段命名

- **前端变量**: camelCase (`orderNo`, `warehouseId`)
- **API 参数**: snake_case (`order_no`, `warehouse_id`)
- **模型字段**: 全小写 (`status`, `sn_code`)
- 转换通过 `convertParamsToSnakeCase` 自动完成

## Git 工作流

- 提交信息: `<type>: <描述>`（feat/fix/docs/chore）
- 推送前确保 local build 通过
- 推送后等待 Cloudflare 构建验证
- SSH 推送: `git config core.sshCommand 'C:/Windows/.../ssh.exe'`

## 文件编辑

> [!danger]
> **禁止**用 Python `rb`/`wb` 编辑 UTF-8 中文 .vue 文件
> **禁止**用 PowerShell `Out-File -NoNewline` 写 .vue 文件
> 正确方式: PowerShell 的 `[System.IO.File]::WriteAllText` 或 `Set-Content` 管道
